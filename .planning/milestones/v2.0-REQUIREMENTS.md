# Requirements: MDViewer v2.0 — Speed & Memory

**Shipped:** 2026-04-16
**Core Value:** Open a markdown file and see beautifully rendered content instantly

### Memory Correctness

- [x] **MEM-01**: WKUserContentController retain cycle fixed — Phase 1
- [x] **MEM-02**: File reading uses `mappedIfSafe` — Phase 2
- [x] **MEM-03**: Mermaid.js loaded via script-src — Phase 4

- [x] **RENDER-01**: True N-chunk progressive rendering — Phase 2

- [x] **LAUNCH-02**: WKWebView pre-warm implemented and profiled — Phase 3

### Window Management

- [x] **WIN-01**: Per-file window position persistence — Phase 5
- [x] **WIN-02**: Window cascading for multi-open — Phase 5

## Out of Scope (Decided During v2.0)

| Feature | Reason |
|---------|--------|
| Dark mode | Conflicts with LaTeX aesthetic |
| File editing | Viewer, not editor |
| File watching / live reload | Quick preview tool, not a working reference |
| Tabs / multi-document interface | One window per file is simpler |
