## Directory Purposes

**`MDViewer/` (source root):**
- Purpose: All Swift source files for the application target
- Contains: 5 Swift files — no subdirectories for source code, no grouping by layer
- Key files: `AppDelegate.swift` (coordinator), `MarkdownRenderer.swift` (core logic), `WebContentView.swift` (WKWebView bridge)

**`MDViewer/Resources/`:**
- Purpose: Bundled non-code assets copied into the app bundle at build time
- Contains: HTML template, JavaScript library, web fonts
- Key files: `template.html` (the entire rendering skin — CSS + JS), `mermaid.min.js` (loaded lazily at runtime)

**`MDViewer/Resources/fonts/`:**
- Purpose: Latin Modern font family in WOFF2 format for the LaTeX-style typography
- Contains: Three WOFF2 files (regular, bold, monospace)
- Referenced by: `template.html` via `@font-face` declarations with relative URLs

**`MDViewerTests/`:**
- Purpose: XCTest unit tests for the rendering layer
- Contains: 2 test files — only `MarkdownRendererTests.swift` has real tests (8 test methods)
- Key files: `MarkdownRendererTests.swift`

**`MDViewer.xcodeproj/`:**
- Purpose: Xcode project definition
- Generated: Yes — via `xcodegen generate` from `project.yml`
- Committed: Yes

**`MDViewer.app/`:**
- Purpose: Pre-built application bundle
- Generated: Yes (build output)
- Committed: Yes (allows direct download without building)

## Key File Locations

**Entry Points:**
- `MDViewer/main.swift`: Bootstrap — `NSApplication.shared`, `AppDelegate()`, `app.run()`
- `MDViewer/AppDelegate.swift`: All app-level coordination

**Configuration:**
- `project.yml`: XcodeGen configuration — deployment target (macOS 13.0), Swift version (5.9), targets, SPM dependencies
- `Package.swift`: SPM manifest declaring `cmark-gfm` and `cmark-gfm-extensions` from `https://github.com/apple/swift-cmark.git` (branch: `gfm`)
- `MDViewer/Info.plist`: Bundle ID (`com.mdviewer.app`), document type registrations, `NSHighResolutionCapable`

**Core Logic:**
- `MDViewer/MarkdownRenderer.swift`: `MarkdownRenderer` class, `SplitTemplate` struct, `RenderResult` struct
- `MDViewer/WebContentView.swift`: `WebContentView` class, `WebContentViewDelegate` protocol

**Rendering Skin:**
- `MDViewer/Resources/template.html`: The entire visual design — all CSS and JavaScript in one file

**Testing:**
- `MDViewerTests/MarkdownRendererTests.swift`: Unit tests for renderer correctness and chunking behavior

## Naming Conventions

**Files:**
- PascalCase matching the primary type they define: `MarkdownRenderer.swift`, `WebContentView.swift`, `MarkdownWindow.swift`
- Test files suffix the tested type with `Tests`: `MarkdownRendererTests.swift`
- Resources use lowercase with hyphens: `template.html`, `mermaid.min.js`, `lmroman10-regular.woff2`

**Types:**
- Classes: PascalCase, `final` by default — `final class MarkdownRenderer`, `final class WebContentView`
- Structs: PascalCase — `struct RenderResult`, `struct SplitTemplate`
- Protocols: PascalCase with semantic suffix — `WebContentViewDelegate`
- No enums present in the codebase

**Functions and Properties:**
- camelCase for methods and stored properties
- `MARK: -` comment blocks used to group methods within a file (e.g., `// MARK: - App Lifecycle`, `// MARK: - Private`)
- Private helpers grouped under `// MARK: - Private`

**Constants:**
- `private static let` for class-level constants: `defaultSize`, `frameSaveKey`, `mermaidRegex`, `blockTagRegex`

## Where to Add New Code

**New rendering feature (e.g., syntax highlighting, footnotes):**
- Primary code: `MDViewer/MarkdownRenderer.swift` — add private methods, update `render(markdown:)` pipeline
- CSS changes: `MDViewer/Resources/template.html` — add styles in the `<style>` block
- Tests: `MDViewerTests/MarkdownRendererTests.swift` — add `testXxx()` methods to `MarkdownRendererTests`

**New UI feature (e.g., table of contents sidebar, zoom):**
- Implementation: `MDViewer/WebContentView.swift` for WKWebView interactions; add JS to `MDViewer/Resources/template.html` for in-page behavior
- Menu wiring: `MDViewer/AppDelegate.swift` — add `@objc` action method, register in `setupMenu()`
- Window chrome: `MDViewer/MarkdownWindow.swift`

**New file format support:**
- Open panel filtering: `MDViewer/AppDelegate.swift` — update `openDocument(_:)` allowed types
- Document type registration: `MDViewer/Info.plist` — add to `CFBundleDocumentTypes`
- Rendering logic: New file in `MDViewer/` following PascalCase naming convention

**New JavaScript capability:**
- Add function to the `(function() { ... })()` IIFE in `MDViewer/Resources/template.html`
- Call from Swift via `webView.evaluateJavaScript("window.yourFunction()")` in `MDViewer/WebContentView.swift`

**Utilities / shared helpers:**
- No dedicated utilities directory — add free functions or extensions directly in the most relevant `.swift` file
- If broadly reused, create a new PascalCase `.swift` file in `MDViewer/`
