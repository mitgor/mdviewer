  <action>
    **Decision gate (per D-01):** Based on the profiling data from Task 1:
    - If WKWebView creation is >20ms on warm launch: IMPLEMENT pre-warm (proceed below)
    - If WKWebView creation is <=20ms: SKIP pre-warm, document rejection in 03-PROFILING.md with evidence, and commit only the profiling document
    - If user typed "skip-profiling": IMPLEMENT pre-warm (industry data shows 50+ms is likely)

    **If implementing pre-warm (per D-02):**

    In AppDelegate.swift:

    1. Add a private property: `private var preWarmedContentView: WebContentView?`

    2. In `applicationDidFinishLaunching`, after `setupMenu()` and before `NSApp.activate`, add:
       ```swift
       preWarmedContentView = WebContentView(frame: .zero)
       ```
       This calls the exact same init as displayResult, guaranteeing configuration match (avoids Pitfall 1).

    3. Modify `displayResult(_:for:paintState:)` to reuse the pre-warmed view for the first file:
       ```swift
       let contentView: WebContentView
       if let preWarmed = preWarmedContentView {
           contentView = preWarmed
           preWarmedContentView = nil  // Only first file gets pre-warmed view
       } else {
           contentView = WebContentView(frame: .zero)
       }
       ```
       The rest of displayResult stays unchanged — delegate assignment, window creation, loadContent, makeKeyAndOrderFront all work the same way.

    4. In `applicationWillTerminate` (add this method if it doesn't exist), nil out the pre-warmed view for clean shutdown:
       ```swift
       func applicationWillTerminate(_ notification: Notification) {
           preWarmedContentView = nil
       }
       ```
       This handles Pitfall 4 (pre-warmed view leaks if app quits without opening a file).

    - Change main.swift — signpost is already there from Plan 01
