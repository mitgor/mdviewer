    1. Add a private instance property `hasCompletedFirstLaunchPaint` (Bool, default false) to AppDelegate.

    2. In `webContentViewDidFinishFirstPaint(_:)`, BEFORE the existing open-to-paint logic, add the launch-to-paint completion guard:

    ```swift
    if !hasCompletedFirstLaunchPaint {
        hasCompletedFirstLaunchPaint = true
        launchSignposter.endInterval("launch-to-paint", launchSignpostState)
    }
    ```

    This ensures the launch-to-paint interval ends exactly once on the very first paint event (per research Pattern 1). The `launchSignposter` and `launchSignpostState` are accessible because they are module-level constants in main.swift (same module).

    3. Handle the "no file opened" case (per research Pitfall 2): In `applicationDidFinishLaunching`, at the END of the method (after the existing DispatchQueue.main.async block), add a delayed guard that ends the launch signpost if no file was opened within 5 seconds:

    ```swift
    DispatchQueue.main.asyncAfter(deadline: .now() + 5.0) { [weak self] in
        guard let self = self, !self.hasCompletedFirstLaunchPaint else { return }
        self.hasCompletedFirstLaunchPaint = true
        launchSignposter.endInterval("launch-to-paint", launchSignpostState)
    }
    ```

    This prevents an infinite signpost interval when the user launches the app without opening a file (e.g., via Cmd+O later).
