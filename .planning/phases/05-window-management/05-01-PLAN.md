**MarkdownWindow.swift changes (only file modified):**

1. Remove `private static let frameSaveKey = "MDViewerWindowFrame"` (the single shared key).

2. Add a static method to derive a per-file autosave name from a file URL:
```swift
private static func autosaveName(for fileURL: URL) -> NSWindow.FrameAutosaveName {
    let path = fileURL.standardizedFileURL.path
    return "MDViewer:\(path)"
}
```

3. Add a static var to track cascade position across window opens:
```swift
private static var lastCascadePoint: NSPoint = .zero
```

4. Rewrite `init(fileURL:contentView:)`:
   - Compute initial frame using `defaultSize` centered on main screen's visible frame (same centering logic as current `cascadedOrigin`).
   - Call `super.init(contentRect:styleMask:backing:defer:)` with this centered frame.
   - Set all existing window properties: title, titlebarAppearsTransparent = false, isReleasedWhenClosed = false, minSize = NSSize(width: 400, height: 300), alphaValue = 0.
   - Set contentView to the passed contentView.
   - Call `setFrameAutosaveName(Self.autosaveName(for: fileURL))`. AppKit automatically restores a saved frame if one exists in UserDefaults for this name.
   - Detect if autosave restored a frame: save `self.frame` before the call and compare after. If frame DID change, autosave restored a position — skip cascading. If frame did NOT change (no saved position), apply cascade: `Self.lastCascadePoint = self.cascadeTopLeft(from: Self.lastCascadePoint)`.
   - Call `configureHighRefreshRate()` at the end, same as before.

5. Remove `loadSavedFrame()` entirely — `setFrameAutosaveName` handles persistence.

6. Remove `cascadedOrigin()` entirely — replaced by `cascadeTopLeft(from:)`.

7. Keep `showWithFadeIn()`, `deinit`, and `configureHighRefreshRate()` unchanged.

**AppDelegate.swift: No changes needed.** The existing `MarkdownWindow(fileURL: url, contentView: contentView)` call already passes the file URL. All persistence and cascade logic is self-contained in MarkdownWindow.
