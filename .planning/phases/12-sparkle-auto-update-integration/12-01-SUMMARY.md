## Decisions Made

Followed all locked decisions from the plan verbatim. Three runtime-only judgments (none overriding the plan):

1. **Root Package.resolved gitignored.** When Swift's CLI sees a repo-root Package.swift it auto-writes a `.build/` and a root `Package.resolved`. The plan's artifact list only mentions the xcworkspace Package.resolved as canonical. Rather than commit both (and risk drift), gitignored the root artifacts so Xcode remains the single source of truth.
2. **No @MainActor annotation on `updaterController`.** Plan Task 4 notes this as a possible fallback if Swift emits actor-isolation warnings. Build emitted **zero warnings** at Swift 5.9 strict-concurrency-off — no fallback needed. Documented here so future Swift 6 upgrade work knows to revisit.
3. **Test-host hang deferred.** `xcodebuild test` hangs in this worktree both with AND without Plan 12-01 changes (verified via `git stash` round-trip). Pre-existing; out of scope per SCOPE BOUNDARY. Logged to `deferred-items.md`.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Gitignore SPM CLI byproducts to prevent untracked-file leakage**
- **Found during:** Task 1 post-Edit (git status showed `.build/` and root `Package.resolved` as untracked after Package.swift was written — Swift's CLI auto-resolves when it sees the manifest)
- **Issue:** Neither the plan nor existing .gitignore accounted for SPM-CLI artifacts at repo root. Leaving them untracked would cause them to appear in every future `git status` and risk a future `git add .` accidentally committing the root Package.resolved, which would drift from the xcworkspace lockfile (the plan's canonical artifact).
- **Fix:** Added `.build/` and `/Package.resolved` to `.gitignore` with an explanatory comment anchoring to the xcworkspace lockfile as the source of truth.
- **Files modified:** `.gitignore`
- **Verification:** `git status --short` after the change showed only the three intended Task-1 files (`.gitignore`, `project.yml`, `Package.swift`).
- **Committed in:** `beba569` (Task 1 commit)

---

**Total deviations:** 1 auto-fixed (Rule 3 — blocking).
**Impact on plan:** Minimal. Keeps the worktree clean and prevents a future accidental commit of the non-canonical root lockfile. Does not alter the plan's artifact set or any task outcome.
