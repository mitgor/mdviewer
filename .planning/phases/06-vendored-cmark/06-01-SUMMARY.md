## Decisions Made
- **Unified modulemap:** Xcode's explicit module compilation caused redefinition errors when `cmark_gfm_extensions` module's header textually included headers from `cmark_gfm` module. Resolved by defining both modules in a single modulemap at `Vendor/cmark-gfm/src/include/module.modulemap`. Swift code now uses only `import cmark_gfm`.
- **CLANG_ENABLE_MODULES: NO for C targets:** C source files include their own headers directly; enabling Clang modules in C compilation units caused the same header to be parsed both textually and as a module, producing redefinition errors. Disabled modules for C targets only; Swift targets still use modules normally.

- **Found during:** Task 2 (Build configuration)

- **Fix:** Unified both module definitions into a single modulemap file; removed separate extensions modulemap; disabled CLANG_ENABLE_MODULES for C targets; consolidated Swift imports to single `import cmark_gfm`
- **Files modified:** Vendor/cmark-gfm/src/include/module.modulemap, Vendor/cmark-gfm/extensions/include/module.modulemap (deleted), project.yml, MDViewer/MarkdownRenderer.swift
