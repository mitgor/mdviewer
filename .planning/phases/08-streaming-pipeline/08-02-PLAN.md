Add a `setRemainingChunks(_:hasMermaid:)` method to WebContentView that enables deferred chunk delivery for the streaming pipeline. This method handles two timing scenarios:

1. **Chunks arrive before firstPaint** (normal case for large files): Store chunks and hasMermaid flag. The existing firstPaint handler will inject them when ready.
2. **Chunks arrive after firstPaint** (possible for very small files where firstPaint fires before onComplete): Inject immediately.

Add the following method to WebContentView, in the public API section (after `loadContent`):

```swift
/// Accept remaining chunks after initial page load (streaming pipeline).
/// If firstPaint has already fired, injects chunks immediately.
/// Otherwise stores them for injection by the firstPaint handler.
func setRemainingChunks(_ chunks: [String], hasMermaid: Bool) {
    self.remainingChunks = chunks
    self.hasMermaid = hasMermaid
    // If firstPaint already fired (small file fast path), inject now
    if hasProcessedFirstPaint {
        injectRemainingChunks()
        if hasMermaid {
            loadAndInitMermaid()
        }
    }
    // Otherwise, the firstPaint handler (userContentController didReceive)
    // will call injectRemainingChunks() and loadAndInitMermaid() as it already does.
}
```

Replace the `openFile` method in AppDelegate to use the streaming pipeline. The key change: instead of calling `renderFullPage` which collects all chunks then dispatches to main thread, call `renderStreaming` which fires `onFirstChunk` mid-callback.

Replace the existing `openFile` method with:

```swift
private func openFile(_ url: URL) {
    guard let tmpl = template else { return }
    let paintState = appSignposter.beginInterval("open-to-paint")
    pendingFileOpens += 1

    // Parse markdown on background thread with streaming dispatch
    let renderer = self.renderer
    DispatchQueue.global(qos: .userInitiated).async { [weak self] in
        let success = renderer.renderStreaming(fileURL: url, template: tmpl,
            onFirstChunk: { page in
                // Called from background thread during C callback --
                // dispatch first chunk to main thread immediately (STRM-01)
                DispatchQueue.main.async { [weak self] in
                    guard let self else { return }
                    // Create window and load first chunk (no remaining chunks yet)
                    let firstResult = RenderResult(
                        page: page,
                        remainingChunks: [],
                        hasMermaid: false
                    )
                    self.displayResult(firstResult, for: url, paintState: paintState)
                }
            },
            onComplete: { remainingChunks, hasMermaid in
                // Called from background thread after all chunks produced
                DispatchQueue.main.async { [weak self] in
                    self?.pendingFileOpens -= 1
                    // Find the window for this URL and deliver remaining chunks
                    if let window = self?.windows.last(where: {
                        $0.representedURL == url
                    }) {
                        window.contentViewWrapper.setRemainingChunks(
                            remainingChunks, hasMermaid: hasMermaid
                        )
                    }
                }
            }
        )

        if !success {
            DispatchQueue.main.async { [weak self] in
                self?.pendingFileOpens -= 1
                appSignposter.endInterval("open-to-paint", paintState)
                let alert = NSAlert()
                alert.messageText = "Cannot Open File"
                alert.informativeText = "The file \(url.lastPathComponent) could not be read."
                alert.alertStyle = .warning
                alert.runModal()
            }
        }
    }
}
```

Key design decisions:
- `onFirstChunk` dispatches to main thread and calls `displayResult` with an empty-remaining-chunks RenderResult. This reuses the existing window creation logic.
- `onComplete` finds the window by URL and calls `setRemainingChunks` to deliver the remaining content.
- `pendingFileOpens` is decremented in `onComplete` (not `onFirstChunk`), so the app doesn't quit while chunks are still being produced.
- Error handling: if `renderStreaming` returns false (file read failure), show the same NSAlert as before.

Check that `MarkdownWindow` exposes `representedURL` or a way to match windows by URL. The existing `MarkdownWindow(fileURL:contentView:)` initializer sets the window title to the filename. Check if `representedURL` is set. If not, use `windows.last` as a simpler approach (streaming opens are sequential in practice), or add `representedURL` to the init.

Look at MarkdownWindow.swift to see if `representedURL` is set. If the window has `representedFilename` or `representedURL` already set, use it. Otherwise, the simplest approach is to use `self.windows.last` since each `openFile` call creates one window, and the `displayResult` call in `onFirstChunk` appends it.

Alternative simpler approach if representedURL is not available: capture the contentView reference from `onFirstChunk` and use it in `onComplete`:

```swift
private func openFile(_ url: URL) {
    guard let tmpl = template else { return }
    let paintState = appSignposter.beginInterval("open-to-paint")
    pendingFileOpens += 1

    let renderer = self.renderer
    var streamContentView: WebContentView?

    DispatchQueue.global(qos: .userInitiated).async { [weak self] in
        let success = renderer.renderStreaming(fileURL: url, template: tmpl,
            onFirstChunk: { page in
                DispatchQueue.main.async { [weak self] in
                    guard let self else { return }
                    let contentView = self.webViewPool.dequeue() ?? WebContentView(frame: .zero)
                    contentView.delegate = self
                    contentView.setNavigationDelegate(self)
                    self.openToPaintStates[ObjectIdentifier(contentView)] = paintState
                    streamContentView = contentView

                    contentView.loadContent(page: page, remainingChunks: [], hasMermaid: false)

                    let window = MarkdownWindow(fileURL: url, contentView: contentView)
                    self.windows.append(window)

                    NotificationCenter.default.addObserver(
                        forName: NSWindow.willCloseNotification,
                        object: window,
                        queue: .main
                    ) { [weak self] notification in
                        guard let closedWindow = notification.object as? MarkdownWindow else { return }
                        self?.windows.removeAll { $0 === closedWindow }
                    }

                    window.makeKeyAndOrderFront(nil)
                    window.alphaValue = 1.0
                }
            },
            onComplete: { remainingChunks, hasMermaid in
                DispatchQueue.main.async { [weak self] in
                    self?.pendingFileOpens -= 1
                    streamContentView?.setRemainingChunks(
                        remainingChunks, hasMermaid: hasMermaid
                    )
                }
            }
        )

        if !success {
            DispatchQueue.main.async { [weak self] in
                self?.pendingFileOpens -= 1
                appSignposter.endInterval("open-to-paint", paintState)
                let alert = NSAlert()
                alert.messageText = "Cannot Open File"
                alert.informativeText = "The file \(url.lastPathComponent) could not be read."
                alert.alertStyle = .warning
                alert.runModal()
            }
        }
    }
}
```

Use this second (simpler) approach. It inlines the displayResult logic to capture the contentView reference. The existing `displayResult` method can remain for any other callers, but the streaming path does not use it.

The `[weak self]` pattern follows the existing codebase convention (see PATTERNS.md: Weak Self in Dispatch Closures).
