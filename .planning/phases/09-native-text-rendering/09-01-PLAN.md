**NativeRenderer.swift:**
Create `MDViewer/NativeRenderer.swift` implementing a cmark AST walker that produces NSAttributedString.

Structure:
```swift
import Cocoa
import os
import cmark_gfm

/// Result of native rendering -- immutable value type matching project conventions.
struct NativeRenderResult {
    let attributedString: NSAttributedString
}

/// Module-level signposter for native rendering instrumentation (PERF-02).
let nativeRenderingSignposter = OSSignposter(
    subsystem: "com.mdviewer.app",
    category: "NativeRendering"
)

final class NativeRenderer {
    ...
}
```

Key implementation details:

1. **Font registration** -- static method `registerFonts()` called once. Uses `CTFontManagerRegisterFontsForURL` with `.process` scope. Register all 3 OTF files. Guard against double-registration with a static `fontsRegistered` bool.

2. **Font caching** -- create all NSFont variants at init time (not per-node). Cache body, bold, italic, bold-italic, code, code-block, and h1-h6 fonts as private properties. Use font name "Latin Modern Roman 10" for serif, "Latin Modern Mono 10" for mono. If font creation fails (name mismatch), fall back to "Times New Roman" for serif and "Menlo" for mono with a debug print warning.

3. **Typography constants** -- match template.html CSS exactly:
   - Body: 16pt, line-height 1.6 (paragraphStyle.lineHeightMultiple = 1.6), color NSColor(red:0.2, green:0.2, blue:0.2, alpha:1) (#333)
   - Headings: h1=32pt, h2=24pt, h3=20pt, h4=17.6pt, h5=16pt, h6=14.4pt, color NSColor(red:0.102, green:0.102, blue:0.102, alpha:1) (#1a1a1a)
   - Code inline: 14.4pt mono, background #f8f6f0
   - Code block: 13.6pt mono, background #f8f6f0
   - Blockquote: italic, color #555
   - Link: color #1a3a6b, .link attribute = URL
   - HR: paragraph with top border via NSAttributedString.Key custom drawing (or use a centered line of "---" in gray)

4. **Style stack** -- maintain a `[StyleState]` stack. StyleState is a private struct holding: font traits (bold/italic), fontSize, textColor, link URL, isCode, listDepth, listType (ordered/unordered), listItemIndex.
   - On CMARK_EVENT_ENTER for container nodes (heading, emph, strong, blockquote, list, item, link): push new state with modified attributes
   - On CMARK_EVENT_EXIT: pop state
   - On leaf nodes (TEXT, CODE, SOFTBREAK, LINEBREAK, HTML_BLOCK, THEMATIC_BREAK, CODE_BLOCK): append to result immediately using current top-of-stack attributes

5. **Node type handling** -- use cmark_node_get_type() for core types. For extension types (strikethrough, tasklist), use cmark_node_get_type_string() and compare the returned C string. Handle:
   - CMARK_NODE_TEXT: append literal with current attributes
   - CMARK_NODE_SOFTBREAK: append " " (space) or "\n" depending on preference (space is standard)
   - CMARK_NODE_LINEBREAK: append "\n"
   - CMARK_NODE_CODE: append literal with mono font + code background color
   - CMARK_NODE_EMPH: push italic trait
   - CMARK_NODE_STRONG: push bold trait
   - CMARK_NODE_HEADING: push heading font size + heading color
   - CMARK_NODE_PARAGRAPH: on EXIT, append "\n\n" with paragraphSpacingBefore
   - CMARK_NODE_BLOCK_QUOTE: push italic + quote color + increased left indent
   - CMARK_NODE_CODE_BLOCK: append full literal with code-block font + background
   - CMARK_NODE_LIST: push list context (ordered vs unordered via cmark_node_get_list_type)
   - CMARK_NODE_ITEM: on ENTER, prepend bullet or number based on list context
   - CMARK_NODE_LINK: push link color + .link attribute with URL from cmark_node_get_url
   - CMARK_NODE_IMAGE: append "[image: alt text]" placeholder (images not supported in NSTextView natively)
   - CMARK_NODE_THEMATIC_BREAK: append horizontal rule line
   - CMARK_NODE_HTML_BLOCK: append raw HTML as plaintext (rare in non-table files)
   - "strikethrough" (via type_string): push strikethrough attribute
   - "tasklist" (via type_string): prepend checkbox character

6. **Paragraph style** -- use NSMutableParagraphStyle with:
   - paragraphSpacingBefore for block spacing (only Before, never both Before and After -- per Pitfall 4)
   - headIndent for list indentation (20pt per level)
   - firstLineHeadIndent for list items
   - lineHeightMultiple = 1.6 for body text

7. **Content width constraint** -- set textContainer.size.width to 680 in NativeContentView (not in renderer). The renderer produces the attributed string without width concerns.

8. **Public API:**
```swift
func render(root: OpaquePointer) -> NativeRenderResult {
    // Walk AST with cmark_iter, return NSAttributedString
}
```
The root pointer is borrowed (caller owns it and will free it). NativeRenderer does NOT free the root.

9. **OSSignposter** -- wrap the render method body with beginInterval/endInterval for "native-render".

10. **Memory safety** -- use `guard let iter = cmark_iter_new(root)` with `defer { cmark_iter_free(iter) }`. The AST root is NOT freed by NativeRenderer (caller's responsibility per Pitfall 5).

Create `MDViewer/NativeContentView.swift` -- an NSView subclass wrapping NSScrollView + NSTextView for native markdown display. Mirror the structure and conventions of WebContentView.

```swift
import Cocoa
import os

/// Delegate protocol for NativeContentView paint completion.
/// Mirrors WebContentViewDelegate for consistent window lifecycle.
protocol NativeContentViewDelegate: AnyObject {
    func nativeContentViewDidFinishFirstPaint(_ view: NativeContentView)
}

final class NativeContentView: NSView {
    weak var delegate: NativeContentViewDelegate?
    private let scrollView: NSScrollView
    private let textView: NSTextView
    private var isMonospace = false
    private var cachedAttributedString: NSAttributedString?
    ...
}
```

Implementation details:

1. **init(frame:)** -- follow WebContentView's init pattern exactly:
   - Create NSScrollView with hasVerticalScroller=true, hasHorizontalScroller=false, autohidesScrollers=true
   - Create NSTextContainer with size NSSize(width: 680, height: CGFloat.greatestFiniteMagnitude) -- 680px matches template.html max-width
   - Set textContainer.widthTracksTextView = false (fixed width for consistent line length)
   - Create NSLayoutManager, add textContainer
   - Create NSTextStorage, add layoutManager
   - Create NSTextView with the textContainer
   - textView.isEditable = false
   - textView.isSelectable = true
   - textView.isVerticallyResizable = true
   - textView.isHorizontallyResizable = false
   - textView.maxSize = NSSize(width: CGFloat.greatestFiniteMagnitude, height: CGFloat.greatestFiniteMagnitude)
   - textView.backgroundColor = .white
   - textView.textContainerInset = NSSize(width: 0, height: 40) -- vertical padding
   - super.init(frame: frame)
   - scrollView.documentView = textView
   - Add scrollView as subview with Auto Layout constraints (pin all 4 edges)
   - Center the text view within the scroll view by setting scrollView.contentView background to white

2. **required init?(coder:)** -- fatalError("init(coder:) not supported")

3. **loadContent(attributedString:)** -- set textStorage content and fire delegate:
   ```swift
   func loadContent(attributedString: NSAttributedString) {
       self.cachedAttributedString = attributedString
       textView.textStorage?.setAttributedString(attributedString)
       // Native rendering is synchronous -- fire paint immediately
       delegate?.nativeContentViewDidFinishFirstPaint(self)
   }
   ```

4. **toggleMonospace()** -- toggle isMonospace flag. For now, this is a no-op placeholder that will be wired when integration happens in Plan 02. The cachedAttributedString is stored so re-rendering can happen when the toggle is connected.

5. **printContent(title:)** -- use NSPrintOperation directly (simpler than WKWebView workaround):
   ```swift
   func printContent(title: String) {
       let printInfo = NSPrintInfo.shared.copy() as! NSPrintInfo
       let printOp = NSPrintOperation(view: textView, printInfo: printInfo)
       printOp.showsPrintPanel = true
       printOp.showsProgressPanel = true
       printOp.run()
   }
   ```

6. **deinit** -- follow project convention:
   ```swift
   deinit {
       #if DEBUG
       print("[NativeContentView] deinit - \(ObjectIdentifier(self))")
       #endif
   }
   ```

7. **Content width centering** -- to center the 680px text container in the scroll view, override `layout()` or use viewDidMoveToWindow to dynamically set the textContainerInset's horizontal component based on the view width: `let hInset = max(20, (bounds.width - 680) / 2)`. Update on resize via overriding `resize(withOldSuperviewSize:)` or using Auto Layout constraints on the textView within the scrollView.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system -> cmark parser | Untrusted markdown content parsed by C library |
| cmark AST -> NSAttributedString | Node literals rendered as attributed text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | NativeRenderer AST walker | accept | Read-only viewer; NSAttributedString sanitizes text content; no script execution unlike WKWebView |
| T-09-02 | Information Disclosure | NativeRenderer link handling | mitigate | Links set as .link attribute on NSAttributedString -- NSTextView opens them via NSWorkspace which validates URLs; no custom URL handling needed |
| T-09-03 | Denial of Service | NativeRenderer large file | accept | Same risk as existing WKWebView path; NSAttributedString handles large content efficiently |
| T-09-04 | Spoofing | Font registration | accept | CTFontManagerRegisterFontsForURL with .process scope -- fonts only visible to this process, no system-wide impact |
</threat_model>
