    **Replace with (per D-05, D-06):**
    ```swift
    private func injectRemainingChunks() {
        guard !remainingChunks.isEmpty else { return }

        let chunkInjectState = renderingSignposter.beginInterval("chunk-inject")
        let chunks = remainingChunks
        let chunkCount = chunks.count
        remainingChunks = []

        for (index, chunk) in chunks.enumerated() {
            let delay = Double(index) * 0.016  // 16ms stagger per D-06
            DispatchQueue.main.asyncAfter(deadline: .now() + delay) { [weak self] in
                self?.webView.callAsyncJavaScript(
                    "window.appendChunk(html)",
                    arguments: ["html": chunk],
                    in: nil,
                    contentWorld: .page
                ) { _ in
                    // End signpost after last chunk injection completes
                    if index == chunkCount - 1 {
                        renderingSignposter.endInterval("chunk-inject", chunkInjectState)
                    }
                }
            }
        }
    }
    ```

    Key details:
    - `arguments: ["html": chunk]` — typed String argument, NO manual escaping (per D-05)
    - `contentWorld: .page` — executes in page JS context where `window.appendChunk` is defined
    - `in: nil` — main frame (no iframes)
    - `[weak self]` in asyncAfter closure — prevents retain cycle during staggered dispatch
    - Signpost `endInterval` moves INTO the completion handler of the LAST chunk (per Pitfall 5 from research)
    - 16ms stagger between chunks via `DispatchQueue.main.asyncAfter` (per D-06)
    - No loading indicators between chunks (per D-07)
    - First paint already happened before this method is called (per D-08, existing flow)

    Do NOT modify any other methods in WebContentView. The `loadContent`, `toggleMonospace`, `loadAndInitMermaid`, `printContent`, `exportPDF`, and `capturePDF` methods remain unchanged.
